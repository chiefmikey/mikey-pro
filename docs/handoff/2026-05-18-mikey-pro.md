## Known Landmines for Next Session

### Memory Gotchas

- **"209 tests all passing" memory is outdated** — was true at v10.2.0 (March), was false at d19419d (March 25), hopefully true again after CI fix lands. Never trust "all passing" without running `npm test` yourself.
- **Memory says "version 9.0.7"** — completely stale. We're at v10.3.x now. Update memory files when bumping versions.

### Setup Gotchas

- **Must run `npm run npm:install` before `npm test`** — sub-package node_modules are NOT in any lockfile. Root `npm install` won't populate them. Forgetting this will cause cryptic "module not found" errors.
- **Sub-packages manage their own dependencies independently** — each `configs/*` directory has its own `node_modules/`. Changes to a sub-package's `package.json` won't auto-apply; must re-run `npm run npm:install`.

### Version Management Gotchas

- **Version drift will resurface if bump-version.js is bypassed** — currently: eslint-config-react at 10.3.2, prettier-config at 10.2.0. If someone edits package.json files manually instead of using `npm run release:bump X.Y.Z`, the drift will come back.
- **Publishing order matters** — `mikey-pro` (unified base) MUST publish first. If a framework config publishes before the base, consumers installing the framework will get an unresolved dependency. Use `npm run publish:all` or publish in this order: mikey-pro, eslint-config, then other scoped packages.

### CI/Workflow Gotchas

- **CI.yml and code-quality.yml are the real gates** — these two workflows are what block PRs. `security.yml`, `dependabot`, and `auto-merge` are independent and green status there does NOT indicate project health.
- **`--max-warnings 0` in code-quality.yml is zero-tolerance** — a single linting warning will fail the entire workflow. Prettier drift (21 files as of this session) is one example.
- **Test timeout is 10s per test** — loading 30+ plugins is heavy. If new plugins are added, test duration may increase further. If tests are timing out at 445s total, individual tests may be close to the 10s limit.

### Prettier/Formatting Gotchas

- **Two theme dirs remain** — `theme/` is in `.prettierignore`, but `theme-adobe/` is NOT. Future cleanup should add `theme-adobe/` to ignore list.
- **Prettier drift is real** — 21 files need formatting as of this session. After CI fix lands, run `npm run fix` to resolve it.

### Release/Publishing Gotchas

- **No v10.x release tags on GitHub** — v10.2.0 was published to npm (March 5) but never tagged on GitHub. Next release (v10.3.3) should create the v10.3.3 tag for consistency.
- **Release notes need manual review** — `scripts/generate-release-notes.js` generates notes from commits. After running it, check that the output is sensible before publishing.
